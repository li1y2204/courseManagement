# courseManagement
一个内存存储的课程管理系统，由go语言来实现
## 技术栈组成
类别 技术/框架 版本 用途 编程语言 Go 1.25.1 主要开发语言 数据存储 内存存储 (Map) - 临时数据存储 数据库驱动 go-sql-driver/mysql v1.9.3 可能是预留的MySQL支持 Web服务 Go标准库 (net/http) - HTTP服务器实现 架构模式 分层架构 - 数据层与Web层分离

## 项目架构
1. 核心架构 ：
   
   - 采用分层架构设计
   - 数据层（data/）：负责数据的CRUD操作，使用内存存储
   - Web层（web/）：处理HTTP请求和路由
   - 公共层（common/）：提供通用功能和常量定义
2. 目录结构 ：
   
   - common/ ：包含公共功能
     - common.go ：通用常量和工具函数
   - data/ ：数据层
     - 按实体划分（teacher、stu、course、class）
     - 每个实体都有对应的CRUD操作
     - 使用内存Map存储数据，支持并发操作
   - web/ ：Web层
     - 处理HTTP请求
     - 路由管理
   - main.go ：程序入口，启动HTTP服务器
3. 数据访问方式 ：
   
   - 使用内存Map存储数据
   - 实现了基本的CRUD操作
   - 使用sync.RWMutex保证并发安全
   - 数据仅保存在内存中，程序重启后数据会丢失
4. Web服务 ：
   
   - 使用Go标准库的 net/http 包
   - 服务器监听在localhost:8080
   - 注册HTTP路由处理函数（通过web包的初始化）
## 技术特点
1. 轻量级 ：使用Go标准库实现Web服务，无额外框架依赖
2. 内存存储 ：数据存储在内存中，适合开发和测试环境
3. 模块化 ：按功能和实体划分目录，代码结构清晰
4. 并发安全 ：使用sync.RWMutex保证数据操作的并发安全
5. 易于理解 ：代码结构简单，逻辑清晰
6. 可扩展性 ：分层架构便于后续功能扩展和维护
